Monitoring services with AWS

Today, after having prodded some features on an app, I realized I forgot to add monitoring to my message queue consumer.

Since we are using AWS, I don’t really know how to health check an application without having a load balancer behind (or, at least, I did not find any way to do that, since the only health check I can have is named EC2).

My first idea was to code some quick program in order to fullfil my needs, quickly, and be done with that, but then I realized that this kind of tool could be useful to anyone out there. So I crafted a little python microservice.

You can find the service here. It only consists of building a cron on top of it, calling the route / as soon as it wants to perform checks. If checks fails and meets alert criterias (hard coded for now), it will trigger an alert.

The documentation is crappy, for now, and it’s only working with SQS, and sending emails if the queue is stuck (a.k.a my queue messages number keeps growing in the last 5 minutes).
I will try to add more stuff as it goes, hoping that other developers, like me, will contribute and add interesting features.
